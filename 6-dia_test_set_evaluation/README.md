# Prospective DIA Case Inference

## Purpose

infer_prospective_cases.py applies a trained Random Forest classifier to prospective DIA peptide datasets and computes peptide‑level evidence scores for amyloid subtype interpretation.

The script performs two main tasks:

1. Predict amyloid subtype probabilities using a trained machine‑learning model.
2. Compute peptide evidence scores for cases with low classifier confidence using diagnostic peptide thresholds.

This provides a hybrid classification strategy combining machine‑learning predictions with interpretable peptide‑level evidence scoring.

The script expects DIA peptide measurements exported from Skyline and a trained classifier generated by train_rf_classifier.py. Feature matrices are automatically aligned to the schema used during model training to ensure compatibility.

## Required Inputs

The script expects the following files and directories in the working directory.

### Skyline-DIA-Reports/

Directory containing Skyline export reports from DIA LC–MS/MS analyses.

Each report must contain the following columns:

- Replicate Name  
- Protein  
- Precursor  
- Total Area Fragment  
- Total Area  
- Isotope Dot Product  
- Library Dot Product  

Expected filename pattern:

DIA-report_*.csv

These reports are typically exported by the user from Skyline using a custom report template.

---

### model_package/

Directory containing the trained classifier generated by train_rf_classifier.py.

Required files:

rf_model.joblib  
Serialized Random Forest classifier.

label_encoder.joblib  
Encoder used to map amyloid subtype labels.

feature_schema.csv  
List of peptide features expected by the trained model.

---

### peptide_thresholds/

Directory containing peptide diagnostic threshold tables generated during DIA peptide processing.

Expected files follow the pattern:

Audit_TYPE.csv

Each table contains diagnostic thresholds and AUROC statistics for peptides associated with a specific amyloid type.

Examples:

Audit_ALL.csv  
Audit_ALK.csv  
Audit_THY.csv

## Output

All results are written to the directory:

output/

Key outputs include:

### prospective_dia_feature_matrix.csv

Feature matrix containing normalized peptide fragment ion intensities for each prospective DIA replicate.

This matrix follows the same feature schema used during classifier training.

---

### rf_predictions.csv

Classifier predictions for each DIA replicate including:

- predicted amyloid subtype  
- classifier confidence score  
- class probability values for all amyloid subtypes  
- indicator of low‑confidence predictions

---

### Peptide Evidence Score Matrices

For each amyloid type, a peptide evidence score matrix is written:

TYPE-feature-score-matrix.csv

Examples:

ALL-feature-score-matrix.csv  
ALK-feature-score-matrix.csv  
THY-feature-score-matrix.csv  

These matrices summarize peptide evidence scores calculated from diagnostic peptide thresholds.

---

### composite-low-confidence-scores.csv

Combined table of peptide evidence scores across amyloid types.

This table supports interpretation of cases where the machine‑learning classifier reports low confidence.

---

## Quality Control Filtering

Peptide measurements are filtered using Skyline quality metrics.

Initial filtering:

DOTP_THRESHOLD = 0.7

Strict filtering:

ISO_MIN = 0.9  
LIB_MIN = 0.8

Additional filtering includes removal of decoy protein entries and zero fragment ion signals.

---

## Classification Workflow

The inference pipeline performs the following steps:

1. Import Skyline DIA reports.
2. Apply quality control filtering.
3. Normalize peptide fragment intensities using run‑level signal normalization.
4. Construct a peptide feature matrix.
5. Align features to the training feature schema.
6. Generate amyloid subtype predictions using the Random Forest classifier.
7. Identify low‑confidence predictions.
8. Compute peptide evidence scores using diagnostic peptide thresholds.

## Run

python infer_prospective_cases.py

---

## Notes

Classifier predictions are based on the peptide feature schema defined during model training. Any missing peptide features are automatically filled with zero intensity values to maintain compatibility with the trained model.

Peptide evidence scoring provides an interpretable secondary assessment of amyloid subtype assignment when classifier confidence is low.
